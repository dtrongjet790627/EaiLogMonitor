## 三、日志解析规则

### 3.1 EAI日志处理流程

```
db trigger get data:[{...}]     ← 1. 触发器数据（包含LINE、PACKID等）
        ↓
kingdee request json: {...}     ← 2. ERP请求
        ↓
kingdee response json: {...}    ← 3. ERP响应（成功/失败）
        或
run error: call lua error:...   ← 3. Lua执行错误
```

### 3.2 字段提取规则

**触发器数据**（`db trigger get data`）：
```
格式: [{"CNT":"16","LINE":"DP EPS1","PACKID":"20251208E2700122","PARTNO":"H25.910.002","WONO":"EPS25120304"}]
```
- LINE：产线（直接使用，不推断）
- PARTNO正则字符类必须包含连字符：`[A-Z0-9.\-]`
  > ⚠️ 历史教训：缺少 `-` 导致 `D24.402.392-M01` 被截断为 `D24.402.392`（2026-03-04修复）

**成功响应**：提取 `SuccessEntitys[0].Number` → `SCHB_NUMBER`

**失败响应**：提取 `Errors[0].Message` → `ERROR_MESSAGE`，清理规则：
1. 去除转义序列 `\r\n` `\n`
2. 截断到中文分号 `；`
3. 截断到反斜杠 `\`

### 3.3 JSON截断处理

EAI系统在约1108字节处截断日志行，触发正则提取路径：
- 日志特征：`JSON不完整，尝试正则提取`
- 处理：`_extract_from_truncated_json()` 逐字段正则提取

## 四、数据库表结构（已验证 2026-03-04）

```sql
CREATE TABLE ACC_ERP_REPORT_SUCCESS (
    ID             NUMBER PRIMARY KEY,
    WONO           VARCHAR2(50),
    PACKID         VARCHAR2(50),
    PARTNO         VARCHAR2(50),
    CNT            NUMBER,
    LINE           VARCHAR2(50),
    SCHB_NUMBER    VARCHAR2(50),
    SOURCE_BILL_NO VARCHAR2(50),
    REPORT_TIME    DATE,
    CREATETIME     DATE DEFAULT SYSDATE,  -- 注意：CREATETIME（无下划线）
    ERROR_MESSAGE  VARCHAR2(2000),
    IS_SUCCESS     NUMBER(1) DEFAULT 1
);
```

> ⚠️ 三个Schema（smt2/dpepp1/dpeps1）均已直接查询确认：列名是 `CREATETIME`，不存在 `CREATE_TIME`

## 五、常见问题排查

> **原则：遇到数据问题，必须SSH查实际日志后再给结论，禁止基于代码推断**

### 5.1 PARTNO字段截断

1. SSH到163，查原始EAI日志确认源头数据
2. SSH到165，查 `eai_monitor.log` 搜索 `JSON不完整，尝试正则提取`
3. 检查 `log_parser.py` 中 `_extract_from_truncated_json` 的正则字符类

### 5.2 ORA-00904列名无效

不要推断，直接查实际表结构：
```python
cursor.execute("SELECT column_name FROM user_tab_columns WHERE table_name='ACC_ERP_REPORT_SUCCESS' ORDER BY column_id")
print([r[0] for r in cursor.fetchall()])
```

### 5.3 产线LINE字段为空

检查 `log_parser.py` 中 `TRIGGER_DATA_PATTERN` 正则，并对比原始日志格式。

### 5.4 服务无法连接EAI服务器

```powershell
ssh -p 2200 root@172.17.10.163 "echo OK"
```

## 六、去重机制

- **成功记录**：基于 `SCHB_NUMBER` 去重，启动时加载最近90天工单号到内存缓存
- **失败记录**：基于 `WONO + PACKID + ERROR_MESSAGE` 组合去重
